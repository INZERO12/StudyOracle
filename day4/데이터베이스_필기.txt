### 데이터베이스
1. 데이터베이스
- 데이터베이스 상태는 수시로 입력/수정/삭제가 수행되기가 자주 변화한다.

 * 데이터베이스 언어
- 데이터 정의어(DDL)
	: 데이터베이스 관리자 + 설계자가 사용(구조 및 관계 설정)
- 데이터 조작어(DML)
	: 데이터베이스 사용자가 데이터에 접근하기 위한 언어정의
- 데이터 제어어(DCL)
	: 데이터베이스 관리자가 사용(권한, 백업 & 복구 등등)
	
	
 * 데이터베이스 사용자
 - DBA(데이터베이스 관리자) : 데이터베이스 전체 환경 구성과 운영 전반의 책임바
 - DB(데이터베이스 설계자) : 요구사항을 분석하여 개념적/물리적 스키마의 설계 책임자
 - 최종 사용자 : 보통 데이터의 입력/수정/삭제/조회에 대한 질의 요청자
 - 시스템 분석가/응용 프로그램 개발자 : 보통 데이터의 입력/수정/삭제/조회에 대한 질의 요청
 
 
 * 데이터베이스 단위
 - 열,속성,애트리뷰드,필드,데이터항목
 - 행,레코트,튜플
 
 * 데이터베이스 추상화
 - DBMS의 주된 역활
  -> 사용자에게 데이터에 대한 추상적인 뷰(View)를 제공
  -> 데이터가 어떻게 저장되고 유지되는지는 숨겨도 무방
 - 3계층 스키마 아키텍쳐
  -> 사용자의 응용프로그램과 물리적 데이터베이스를 분리시키는 것이 중요하다.
  2T(server - client)
  3T(server - 응용프로그램 - client)
  NT(server - 응용프로그램 n개 - client) 
  
2. SQL -> 구조적 질의 언어
- 사용자, 관리자, 설계자 등과 데이터베이스 간의 대화에 필요한 소통 언어.

 * SQL 문장들의 종류
- DML : SELECT, INSERT, UPDATE, DELETE
- DDL : CREATE, ALTER, DROP
- DCL : GRANT, REVOKE
- TCL : COMMIT, ROLLBACK

 * Table : 여러 종류의 Data들을 모아 놓을 수 있는 최소한의 공간
